# Cabinet Manifesto

Cabinet exists to make AI-native work feel whole again.

Today, building with AI often means working across too many disconnected surfaces: terminals, markdown files, chat windows, admin tools, notes, and separate product repos. The work may be powerful, but the experience is fragmented. The human ends up acting as the glue between systems.

Cabinet is built on a simple belief: the future of work needs a single place where the whole system can be seen, understood, and operated.

## The Core Belief

AI workflows should not feel like scattered power tools.

They should feel like a complete environment:

- one place to see projects
- one place to see agents
- one place to see knowledge
- one place to see running work
- one place to understand what is happening across the system

The goal is not just more capability. The goal is coherence.

## A Human Interface For Agentic Systems

Markdown files, terminals, scripts, and agent frameworks are powerful, but they are not enough on their own.

They are often agent-oriented, not people-oriented.

Cabinet is meant to bridge those worlds:

- keep the power and flexibility of agentic workflows
- add a visual, understandable interface around them
- let people work with AI systems without needing to think like infrastructure

The right abstraction is not "hide the system."

It is "make the system legible."

## One Cabinet, Many Drawers

A cabinet is not a single agent.

It is a structured collection of capabilities that work together:

- agent teams
- workflows
- knowledge bases
- files
- visual interfaces
- live execution surfaces

Each cabinet should feel like a ready-to-use operational unit for a real goal.

Not a bag of prompts.
Not a pile of tools.
A working system.

## From Tools To Ready-Made Teams

Most people do not want to assemble a workflow from scratch every time.

They do not want to install five tools, wire together skills, manage disconnected repos, and invent the operating model themselves.

Cabinet should make it possible to start from a complete setup:

- choose a goal
- load a cabinet
- get the agents, workflows, knowledge, and UI that fit that goal

Over time, this should grow into a library of reusable cabinets and eventually a marketplace of complete AI-native operating systems.

## Open Source Is A Product Principle

Cabinet should be open source not just because it is generous, but because it is structurally correct.

Open systems are easier to inspect, adapt, trust, and extend.

If Cabinet is the place where people organize their agents, workflows, and knowledge, then they should be able to see how it works and shape it to fit their own needs.

The community should be able to reproduce it, remix it, and improve it.

## Your Data Should Stay Yours

The system should be built around files people can keep, read, move, and understand.

Knowledge should not be trapped inside opaque tools or locked into a single hosted product.

Cabinet should sit above any one model, provider, or coding assistant.

It should remain useful regardless of which model is popular, which agent framework is in fashion, or which toolchain a person prefers.

The user should own the structure.
The user should own the history.
The user should own the data.

## Local-First, Model-Agnostic, Durable

Cabinet should not depend on one cloud tool or one AI interface to make the system usable.

It should be durable across tool changes.
It should be portable across workflows.
It should be resilient across model shifts.

That means:

- local-first where possible
- simple underlying primitives
- compatibility with multiple agents and model providers
- a system layer that outlives any single AI product

## Simplicity On The Surface, Depth Underneath

Cabinet should feel simple even when the system behind it is sophisticated.

The interface should be approachable enough for technical builders and clear enough for non-technical users.

The standard is not whether power users can force it to work.

The standard is whether ordinary people can understand it, trust it, and use it as naturally as they use their most familiar software today.

## What Cabinet Is Really Trying To Do

Cabinet is an attempt to turn AI work from scattered sessions into organized systems.

It is a place where:

- agents are not isolated
- knowledge is not lost
- workflows are not hidden
- projects are not fragmented
- the human is not forced to be the memory layer

The long-term vision is simple:

people should be able to build and run serious AI-powered operations from one coherent environment.

That is the philosophy.
